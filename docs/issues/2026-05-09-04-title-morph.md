## What to build

Replace the static title rendering in `HomeView`'s unified scroll stack (delivered in Issue 03) with a scroll-driven counter-direction wipe between the split form (Section 1: `BENZ TRA / NSPORT`) and the de-spaced form (Section 2: `BENZTRANSPORT`). The morph fires once at the section boundary as the user scrolls past it; before and after, both halves scroll naturally with their sections.

This slice introduces two pure functions (with unit tests), one new hook (with unit tests), and one new component.

**`src/lib/title.js`** — pure function `splitTitleForMorph(text: string): { split: string, despaced: string }`. Codifies the title-shape rules currently scattered across `MassiveTitle.splitToRows` (slash separator, first-space split, midpoint split for short single-word titles) and `ProjectDetail.jsx:131` (de-spacing via `replace(/[  ]+/g, '')`). The `split` form preserves the line-break grammar today's `MassiveTitle` consumes; the `despaced` form is the single-line version used at the strip section's title slot. Both `MassiveTitle.jsx`'s splitter and the new `MorphingTitle` component (below) consume this single helper.

**`src/lib/clipPath.js`** — pure function `clipPathFromProgress(progress: number, direction: 'wipe-out-right' | 'wipe-in-left'): string`. Returns an `inset(0 X 0 Y)` string. `wipe-out-right` maps `progress: 0 → 1` to `inset(0 0 0 0) → inset(0 0 0 100%)`. `wipe-in-left` maps `progress: 0 → 1` to `inset(0 100% 0 0) → inset(0 0 0 0)`. Clamps `progress` to `[0, 1]`.

**`src/hooks/useScrollProgress.js`** — `useScrollProgress({ ref, enterAt = 0, exitAt = 1 }): number`. Returns a `0..1` progress value as the ref's bounding rect crosses the viewport. `enterAt` and `exitAt` are viewport-fraction thresholds (e.g., `enterAt: 0.6` = "starts measuring once the ref is 60% of the way up the viewport"). Reads from native `scroll` events on `window` (Lenis integration in Issue 05 will replace the listener with a Lenis subscription via the same internal API). Returns `1` immediately under `prefers-reduced-motion: reduce` so consumers render the post-morph state without animating. Cleans up its scroll listener on unmount.

**`src/components/MorphingTitle.jsx`** — `MorphingTitle({ text, ink, boundaryRef })`. Internally:

1. Calls `splitTitleForMorph(text)` once.
2. Calls `useScrollProgress({ ref: boundaryRef })` for the section-boundary progress.
3. Renders both forms stacked at the section boundary slot:
   - Split form (top), `clipPath` driven by `clipPathFromProgress(progress, 'wipe-out-right')`. Wraps the existing `<MassiveTitle text={split} ink={ink} />`.
   - De-spaced form (bottom), `clipPath` driven by `clipPathFromProgress(progress, 'wipe-in-left')`. Wraps `<MassiveTitle text={despaced} ink={ink} animate={false} />` (no cascade — it's positioned and clip-pathed; the cascade already played on selection).
4. Under reduced motion, both `clip-path` values snap to their end states (`progress: 1`) so the de-spaced form is visible and the split form is invisible without any animation.

**Wire `MorphingTitle` into `HomeView`'s selected stack** — replace the static titles introduced in Issue 03 (split title at top of Section 1, de-spaced static title at top of Section 2) with a single `MorphingTitle` instance positioned at the section boundary. The `boundaryRef` points at a sentinel element placed at the bottom of Section 1 (or equivalently, the top of Section 2).

**Update `MassiveTitle.jsx`'s split** to consume `splitTitleForMorph` from `src/lib/title.js` instead of the inline `splitToRows` helper. Delete the inline `splitToRows`. No behavior change — the new pure function returns `split` in the exact shape the existing `splitToRows` did (string with optional newline split via the `/` rule), so `MassiveTitle` continues to render two rows correctly.

**Unit tests** — four test files:

- `src/lib/title.test.js` — `splitTitleForMorph`: explicit `/` separator, first-space separator, midpoint split for `length >= 4` no-space titles, short single-word titles (`length < 4`), empty string, single character, mixed unicode whitespace (` `).
- `src/lib/clipPath.test.js` — `clipPathFromProgress`: both directions at `progress: 0`, `0.5`, `1`; clamps `progress < 0` to `0` and `progress > 1` to `1`; exact `inset(...)` output strings.
- `src/hooks/useScrollProgress.test.jsx` — mount with a ref whose `getBoundingClientRect` is stubbed; assert `0` before viewport entry, `1` after viewport exit, linear `(0, 1)` between; assert reduced-motion shortcut returns `1` immediately. Use `renderHook` + `act` patterns from `useUrlSync.test.jsx`.
- (No test for `MorphingTitle` itself — it's covered by visual regression in Issue 03's regenerated snapshots, plus the boundary cases its dependencies handle.)
